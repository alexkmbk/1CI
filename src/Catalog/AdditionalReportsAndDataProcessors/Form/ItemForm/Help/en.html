Inserting additional reports and data processors.

Some of the data in additional report or data processor cards is filled in automatically. This data was specified by developers in external data processor or report files and cannot be modified (command list, additional information). Any additional data processor can support one or several commands available in applied object windows or in application sections.

Depending on their purpose and settings, additional reports and data processors can contain multiple tabs. In this case, mandatory commands are located on a separate Commands tab. Names and content of other tabs may vary. Special commands can also be located on additional tabs.

Two types of additional reports and data processors are available:

  • Assignable - intended for use with one or several specific application lists or documents. The list of objects to which an additional report or data processor can be assigned is predefined by the developer and specified in the text of the additional report or data processor. When inserting an additional report or data processor, you can only select them from the list.
  • Global (or common) - containing commands that can only be called from command interface sections. The commands are independent of any applied objects. The list of sections where commands can be placed is predefined by the developer, included in additional report or data processor file, and cannot be modified.

When inserting an additional report or data processor, you need to specify its primary parameters:

  • Description - provided by developer and filled in automatically. Can be modified if necessary.

  • Publication radio button - controls report or data processor availability. Can be used to limit access to reports or data processors. You can make a report or data processor available for anyone, available for administrator only, or unavailable. This parameter can take on the following values:

    • Used - data processor or report is available for use. Data processor commands are available for all application users with the Use additional reports and data processors role assigned.
    • Debug mode - data processor or report is only available to users with the Add and edit additional data processors or Full access role assigned (marked in red in the list). For all other users, these commands are hidden. We recommend selecting this mode when an additional report or data processor requires validations.
    • Disabled - data processor or report is unavailable and cannot be selected from the list. We recommend selecting this mode when you need to make a data processor or report temporarily unavailable while keeping all its settings intact. To make a data processor or report available again, select any other mode.
  • Group - to add an additional data processor or report into a list group, click . Select a group from the list. Using groups makes it easier to work with additional reports and data processors.
  • Responsible - you can assign a user to be responsible for additional report or data processor status monitoring, updates, etc. Administrator is assigned as Responsible by default.
  • Comment - enter detailed description here.

Loading from file

To update a data processor in internal application storage, use the Load from file command. Specify file path and click Open.

Saving to file

To export a data processor or report from storage to computer file, use the Save to file command. Specify folder and click Save. File name will be generated automatically but can be changed if necessary.

Working in safe mode

As a rule, data processors should be executed in safe mode. This is a safeguard against untrusted code being executed on server side. However, certain data processors may require execution in unsafe mode. In this case, a warning will be displayed in the additional report or data processor window. These data processors should only be obtained from trusted sources (vendors) or manually audited before execution.

For more information on safe mode, refer to the 1C:Enterprise platform documentation.

Notes on inserting assignable additional reports or data processors

For additional reports and data processors intended for work with application lists or documents, you can select objects for which data processing or report generation commands should be available.

  • Assignment  - to select applied objects, click the hyperlink that lists the selected objects or displays Undefined when no objects are selected. Select the required objects via check boxes.
  • Show - data processing and report generation commands are accessible from:
    • the list of selected objects, if the In lists check box is selected; 
    • the applied object window, if the In catalog and document item forms check box is selected. The check boxes are not mutually exclusive and can be both selected.
  • Commands - the command list is filled in on the basis of the list prepared by developers, by default. These commands will become available in the specified list or document after the additional report or data processor is added.

Notes on inserting global (common) additional reports and data processors

For each report or data processor accessible from command interface (not attached to a configuration object), you need to specify interface sections where the processing commands will be available via the Additional reports and Additional data processors commands. Command list and settings can be located on a separate Commands tab.

  • Placement - to select interface sections, click the hyperlink that lists the selected sections or displays Undefined when no sections are selected. Select the required sections via check boxes and click OK to confirm the selection.

For each command, you can set up the following:

  • In the Quick access column, click  (or press F4) and select the users that should have access to this command by default. The default value is None. Add users to the list and click OK to confirm selection and return to the setup procedure.
  • In the Schedule column, for each global report/data processor command executed on server side, click  and specify the execution schedule for running this command as a scheduled job. Commands with the Open form execution option enabled cannot be scheduled.
