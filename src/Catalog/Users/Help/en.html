This list is used to manage users of the application.

You can access the user list via the Users command in Administration - User and rights settings or Access management section.

Only administrator and user list manager can edit the user list.

Application logon cannot be performed without administrator's approval.

User list manager can edit user list (to a limited degree), and configure users prior to their logon approval by administrator. User list manager also can edit Users and User groups lists.

All other users are only allowed to view their own personal data, edit their contact information, change their password, or select application interface language. 

Users created by the user list manager are assigned Prepared status. Application adiminstrator can easily identify external users by this parameter, verify accuracy of the entered data, and permit application logon.

Once the administrator approves logon for an external user, the user list manager will be unable to change their logon settings in any way, with the exception of revoking the logon approval.

In a distributed Infobase, application logon is configured for each infobase separately.

Lists containing large numbers of users can be grouped for convenience. For example, users can be divided into groups in accordance with corporate organizational structure.

Creating users

  •  Click Create and fill in the required fields.

Available commands

  • User list management commands are available via context menu accessed by right-clicking.
  • Any user can be marked as "not valid". To view the list of these users, click All actions - Show not valid users.
  • If you need to check user data accuracy or modify incorrect user data, use the All actions - Infobase users command to check records or synchronize user list and Infobase user list.

Grouping users

If the application supports user groups, the list will consist of two sections: groups listed on the left, and users belonging to the selected group - on the right. The All users group is specified as current group by default (all application users are displayed).

A user can belong to multiple groups. To add a user to a group, you can use any of the following methods:

  • When creating a user, select a group from the list and click Create.
  • To quickly add multiple users to a group, open the group window and click Pick. The users will be picked from the Users list containing three sections.
  • Select a user in the user list, execute the All actions - Assign groups command, and check the corresponding check boxes to include the user into one or more groups. This command is also available via context menu accessed by right-clicking.
  • Drag the users to the corresponding group. Click Yes to confirm. Before using this feature, you need to clear the Show users of subordinate groups check box.

Creating user groups

  • To create a group, click  Create group.
  • You can also use the Create command available via context menu accessed by right-clicking the left section of the list.

Moving user groups to other groups

  • Select a group in the left section of the list, and use the Move to group command available via context menu accessed by right-clicking.
  • Select a group into which you want to move the current group.

Viewing users in subordinate groups

To display all users in the nested hierarchical groups, select the Show nested group users check box at the bottom of the list. The check box is selected by default.

Picking user group members

When selecting users from the list, the list consists of three sections.  Groups are displayed on the top left, users belonging to the selected group - on the bottom left, and selected users - on the right.

  • For your convenience, available user groups are listed in the top left section. Once you select a group (or the All users group) from this section, all members of this group will be listed in the bottom left section.
  • Select one or more external users from the bottom left section and click the arrow button to move them to the Selected users list on the right.  You can also double-click any user to move them to the Selected users list.
  • To remove external users from the Selected users list, also select the users and click the arrow button, or double-click each external user.
  • Click double arrow button << to remove all users from the Selected users list.
  • The number of users currently in the Selected users list is displayed.
  • Once you have finished picking group members, click End and close.

Viewing User information report

This report is used to analyze logon settings (Infobase user properties) together with other user properties from catalog. 3 report options are available - information on users, external users, or all application users.

  • Click All actions - User information, and generate the report.
