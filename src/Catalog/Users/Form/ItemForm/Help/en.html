Describing user properties.

To enter or edit user data, you need administrator-level rights or user list management-level rights. 

Application logon cannot be performed without administrator's approval. 

User list manager can edit user list (to a limited degree), configure users prior to their logon approval by administrator, edit Users and User groups lists, and edit user access rights.

All other users are only allowed to view their own personal data, edit their contact information, change their Password (if permitted by administrator settings), or select application interface language.

In a distributed Infobase, application logon is configured for each infobase separately.

Recommendations on creating logon passwords

A strong password should:

  • not be human-readable
  • be at least 8 characters long
  • include:
    • uppercase Latin letters
    • lowercase Latin letters
    • numeric characters
    • special characters (underscore, brackets, etc)

The password should not be similar to user name, contain numeric characters only, actual words, or periodic character sequences.

  • Good password examples:
    “nj7{jhjibq*Gfhjkm,
    F5”njnGhkmNj;t{HI.
  • Bad password examples:
    Smith,
    qwerty,
    12345678,
    123123123.

Creating users (only for users with sufficient rights)

When the user list manager creates a new user and specifies their logon name, administrator will be recommended to verify logon settings. Once the administrator approves logon for this user, the recommendation will be hidden and the user list manager will be unable to change the logon settings in any way, with the exception of revoking the logon approval. To view the list of users created by the manager, use filter by the Prepared by field. User data is broken into two tabs:

  • Main properties 
    • Enter Full name of user. This name will be displayed in a variety of application lists and reports. 
    • The Not valid check box is cleared for all users by default. If a user is no longer working with the application for any reason (for example, they resigned), yet cannot be deleted (for example, due to being referenced by other applied objects), it is recommended that you select the Not valid check box for them. Not valid users are not displayed in the user list, and they cannot be selected, for example, as task performer or person responsible for a document. If you need to restore the user to valid status (for example, they have been rehired), simply clear this check box.
    • Name (for logon) - short name used for application logon. Not required to be identical to the full name (a user can have short name "Smith" and long name "John Smith").
    • 1C:Enterprise authentication - by default, check box is selected and password field is active.
    • Password - enter application logon password. This field must not be blank for security purposes. 
    • Password confirmation - enter the password again to make sure it was typed correctly.
    • Show in list - select this check box if you want this user to be displayed in the selection list available at application start. If any external users are registered in the application, selecting this check box is not recommended. If this check box is cleared, this user will not be displayed in the selection list available at application start.
    • User cannot change password - select this check box to block a user from changing their password, if necessary.
    • OpenID authentication - select this check box to allow a user to use unified logon protocol OpenID. This option is used when working with 1C:Enterprise web client. In this case, 1C:Enterprise authentication check box must also be selected for the application to operate correctly.
    • OS authentication - select this check box for quick and easy application logon.
      • in User field, click button  to select user name to be used for OS logon. This name will be also used for application logon. In this case, you can uncheck 1C:Enterprise authentication check box.
    • Start mode - filled automatically by default (Auto). This mode is recommended.

  • Addresses, phones

    • Enter user contact information required by the application (phone number, email address, etc).

Adding users to groups

If the application supports user groups, you can add the user to one or several user groups immediately during user creation procedure.

  • Click Groups on navigation panel. Select one or several groups using check boxes.
  • To clear all selections quickly, click Exclude from all groups.

User access rights 

You can use one of the following methods to set up access rights:

  • Click Access rights on navigation panel to add a user to one or more access groups. Select the required groups from list.
  • In the Allowed actions (roles) list, you can assign one or several roles to a user, thus defining their personal access rights. Some roles may correspond to job functions or duties, some - to more specific functions. 
    • Select the roles you need to assign via check boxes.
    • To hide roles you do not need, click Selected roles only.
    • For your convenience, roles can be sorted alphabetically or By subsystems.

Allowing application logon (for adiminstrator only)

  • To allow a user to logon to the application, select the Access to infobase allowed check box. and provide the required information: logon name, password, and other authentication parameters.

    Managing user settings

    Individual settings of each user are saved by the application for further use. To view, copy, or clear user settings (clearing may be required, for example, when invalid settings prevent normal application use), click Settings on navigation panel.

    Application warnings

    If user properties differ from the Infobase user properties, the application will display a warning including the list of differences. This list should be saved in order to reconcile the differences.
