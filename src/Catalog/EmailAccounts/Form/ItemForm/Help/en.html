Description of email account properties.

When creating an email account, the account parameters are mostly set up automatically, in accordance with the mail server settings.

When creating a system email account, we recommend that you select the Remember password option so that email messages from this account can be sent without human intervention. However, you are free to choose whether to store any account passwords in the infobase or not.

Main additional data

  • Enter account Description. The name should reflect the goal and purpose of the account or account owner (example: Support service). This name is how the account will be represented in a variety of lists.
  • Enter account Email address in the user@mailserver format.
  • Specify Sender name. This name is how the account owner will be represented in the sent emails. Sender name can differ from the user name registered in the application (example: user name = Administrator, sender name = Mr. Johnson).
  • Specify Account usage options:
  • Select the Sending messages check box;
  • Select the Receiving messages check box.

Sending emails (outgoing mail server setup)

  • Enter account Email address in the user@mailserver format. This address must differ from the primary address if you want it to be dedicated to sending email only.
  • Enter account Password if necessary (otherwise, you will be prompted to enter it every time you send an email from this account).
  • Select the Secure password authentication check box if necessary.
  • In the Server field, specify the address of your SMTP server. Example: smtp.gmail.com.
  • Enter computer Port number to be used for outgoing email. Example: 465.
  • Select the Use secure connection (SSL) check box.
  • Select the Mandatory server login before sending email check box if necessary.

Receiving emails (incoming mail server setup)

  • Enter account Email address in the user@mailserver format. This address must differ from the primary address if you want it to be dedicated to receiving email only.
  • Enter account Password if necessary (otherwise, you will be prompted to enter it every time you check this account for incoming email).
  • Select the Secure password authentication check box if necessary.
  • In the Server field, specify the address of your POP3 server. Example: pop.gmail.com. 
  • Select Protocol to be used for the incoming mail server: POP or IMAP.
  • Enter computer Port number to be used for incoming email. Example: 993.
  • Select the Use secure connection (SSL) check box.
  • Select the Keep email copies on server check box if you do not want the incoming messages that you just downloaded to be deleted from the mail server. This check box is cleared by default, forcing all messages to be deleted from the mail server as soon as you download them to a local computer.
  • In the Delete messages from server after field, specify how long you want the messages to be stored on the mail server. Example: 10 days.

Other settings

  • Specify Server timeout. Example: 30 sec.

Validating settings

  • Click Validate settings to validate email account parameters.

  • This will send a standard email message in order to check that connection to incoming mail server can be established (provided that the email account is used to receive email). You can also use the All actions - Validate settings command to the same effect.
