External user groups

Viewing and editing properties of external user groups.

External users can be combined into groups based on a variety of parameters. An external user group can include users of the same type, such as counterparties or individuals. You can also create groups that include external users sharing a common property, such as survey respondents. This approach facilitates access rights setup.

Creating external user groups

  • Enter group Description.
  • In the Group members field, you can specify type of the group members. The group can include:
    • external users of any type - for this, set Any users value;
    • external users of a specific type (example: counterparties) - for this, set Any users of specified type value and specify a member selection list.
    • all external users of a specific type (example: all counterparties) - for this, set All users of specified type value and specify a member selection list. In this case, Pick field will be disabled. All external users from the list will be added to the group by default.
  • When editing the list of external user group members, keep in mind that these users also belong to the parent groups.
  • To add a large number of external users quickly, use Pick command. This allows selecting multiple external users and adding all of them to a group.
  • The Add command only allows adding a single external user at a time.
  • The new group can be made subordinate to another group. For this, select the required group in the In group field list.
  • Comment field can be used to enter a detailed description of the group.

External user group access rights

You can use one of the following methods to set up access rights:
  • Click Access rights on group navigation panel. Click Include in group, select an access group from list.
  • If roles are assigned directly, you can assign roles both to single external users and to external user groups.
