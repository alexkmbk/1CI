This list is used to manage user groups.

User groups can be convenient when a large number of users work with the application:

  • to group large numbers of users (for example, during user selection);
  • to quickly set up access rights for all users in a group (example: if you add all accountants to Accountants group, you can set up access rights to lists and documents for the whole group at once, not for each accountant separately);
  • to set up other group-level parameters, such as business process task performer roles, etc.

Groups are created as a flat list (all groups are independent), or as a hierarchical structure with parent and subordinate groups. In group hierarchy, all members belonging to a subordinate group also belong to the parent groups.

Predefined group All users contains all application users. This group is specified as current group by default.

If roles are assigned directly, you can assign roles both to single users and to user groups.

Grouping users

You can create user groups containing users of a specific type. This approach facilitates access rights setup. For example, users can be divided into groups in accordance with corporate organizational structure, job titles, or project team membership:

  • Enterprise
    • Administration
      • Department 1
      • Department 2
    • Administration directors
    • Engineering and technical services
      • Department 1
      • Department 2
        • Group 1
        • Group 2
      • Engineering and technical service directors
    • Manufacturing
      • Manufacturing shop 1
        • Shop area 1
        • Shop area 2
      • Manufacturing shop 2
        • Shop area 1
        • Shop area 2
        • Shop area 3
      • Manufacturing directors
    • Enterprise directors
    • Joint inventory group
