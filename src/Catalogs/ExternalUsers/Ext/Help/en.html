This list contains accounts of "outside" users working with the application. Generally, access rights granted to these users are very limited. External users do not actively input data, work separately, and do not interact with each other. Examples of external users: employees in HR database, online store customers, questionnaire responders, etc.

You can access the external user list via the Administration - User and rights settings section. It can also be accessed via section Questioning (Access management).

External users of different types can access the same application; therefore, information on external users can be distributed between multiple application lists.

Only administrator and external user list manager can edit the external user list.

Application logon cannot be performed without administrator's approval. 

External user list manager can edit External users list (to a limited degree), configure external users prior to their logon approval by administrator. The list manager can use External access command, edit External users and External user groups lists.

External users created by the external user list manager are assigned Prepared status. Application adiminstrator can easily identify external users by this parameter, verify accuracy of the entered data, and permit application logon.

Once the administrator approves logon for an external user, the user list manager will be unable to change their logon settings in any way, with the exception of revoking the logon approval.

External users are only allowed to view their own personal data, or edit personal parameters such as their password, application interface language, or contact information.

In a distributed Infobase, application logon is configured for each infobase separately.

Creating external users

  • External access command is the primary tool for creating and modifying external users. This command is located in the command bar of each corresponding applied object or list. The command creates a new external user linked to the selected object, or displays information on an existing external user.
  • You can also create an external user directly in the user list. Click Create. In this case, you will need to specify Object linked to the external user you want to create.

Available commands

  • Any external user can be marked as "not valid". To view the list of these external users, click All actions - Show not valid external users.
  • If you need to check external user data accuracy or modify incorrect external user data, use command All actions - Infobase users to check records or synchronize external user list and Infobase user list.

Grouping external users

If the application supports user groups, the list will consist of two sections. Groups are displayed on the left, external users - on the right. When you select a group, all external users belonging to this group will be displayed in the right section.

Long lists of external users can be grouped for your convenience. Grouping can be performed, for example, based on user relation types: partners, counterparties, etc. 

An external user can belong to multiple groups. To add an external user to a user group, you can use any of the following methods:

  • When creating an external user, select a group from the list and click Create.
  • To quickly add multiple external users to a group, open the group window and click Pick. The users will be picked from the External users list containing three sections.
  • Select an external user in the user list, use the All actions - Assign groups command to include the user into one or more groups. This command is also available via context menu accessed by right-clicking.
  • Drag one or more external users to a group. Before using this feature, you need to clear the Show external users of subordinate groups check box.

Creating external user groups

  • Click   Create group and fill in the required fields.
  • You can also use the Create command available via context menu accessed by right-clicking the left section of the list.

Moving external user groups into other groups

  • Select a group in the left section of the list, and use Move to group command available via context menu accessed by right-clicking.
  • Select a group into which you want to move the current group.

Viewing external users in subordinate groups

  • To view all external users belonging to the subordinate groups in the hierarchy, select the Show external users of subordinate groups check box at the bottom of the list. The check box is selected by default.

Picking external user group members

When selecting external users from the list, the list consists of three sections.  Groups are displayed on the top left, external users belonging to the selected group - on the bottom left, selected users - on the right.

  • For your convenience, available external user groups are listed in the top left section. Once you select a group (or the All external users group) from this section, all members of this group will be listed in the bottom left section.
  • Select one or more external users from the bottom left section and click the arrow button to move them to the Selected users list on the right.  You can also double-click any external user to move them to the Selected users list.
  • To remove external users from the Selected users list, also select the users and click the arrow button, or double-click each external user.
  • Click double arrow button << to remove all external users from the Selected users list.
  • The number of external users currently in the Selected users list is displayed.

Viewing the External user information report

This report is used to analyze logon settings (Infobase user properties) together with other external user properties from catalog. 3 report options are available - information on users, external users, or all application users.

  • Click All actions - User information, and generate the report.
