Describing external user properties.

To enter or edit external user data, you need administrator-level rights or external user list management-level rights. 

Application logon cannot be performed without administrator's approval.

External user list manager can edit the External users list (to a limited degree), configure external users prior to their logon approval by administrator, use the External access command, edit the External users and External user groups lists, and edit external user access rights.

All other users are only allowed to view their own personal data, edit their contact information, change their Password (if permitted by administrator settings), or select application interface language.

In a distributed Infobase, application logon is configured for each infobase separately.

Recommendations on creating logon passwords

A strong password should:

  • not be human-readable
  • be at least 8 characters long
  • include:
    • uppercase Latin letters
    • lowercase Latin letters
    • numeric characters
    • special characters (underscore, brackets, etc)

The password should not be similar to user name, contain numeric characters only, actual words, or repeating character sequences.

  • Good password examples:
    “nj7{jhjibq*Gfhjkm,
    F5”njnGhkmNj;t{HI.
  • Bad password examples:
    Smith,
    qwerty,
    12345678,
    123123123.

Creating external users (only for users with sufficient rights)

When the external user list manager creates a new external user and specifies their logon name, administrator will be recommended to verify logon settings. Once the administrator approves logon for this external user, the recommendation will be hidden and the external user list manager will be unable to change the logon settings in any way, with the exception of revoking the logon approval. To view the list of external users created by the manager, apply filter to the Prepared by field.

  • Each external user is linked to an applied object (counterparty, employee, etc) specified in the Object field. When creating external users via the External access command (recommended), the Object field is filled in automatically.
  • When creating external users in the external user list via the Create command, the Object field must be filled in manually. For this, select an application list via the Show all command, and select a list object linked to the new external user.
  • Enter the Full name of external user. This name will be displayed in a variety of application lists. 
  • The Not valid check box is cleared for all external users by default. If an external user is no longer working with the application for any reason, yet cannot be deleted (for example, due to being referenced by other applied objects), it is recommended that you select the Not valid check box. The Not valid external users are not displayed in the external user list, and they cannot be selected.
  • Name (for logon) - short name used for application logon.
  • 1C:Enterprise authentication - by default, this check box is selected and password field is active.
  • Password - enter application logon password. This field must not be blank for security purposes. 
  • Password confirmation - enter the password again to make sure it was typed correctly.
  • User cannot change password - select this check box to block an external user from changing their password, if necessary.
  • OpenID authentication - select this check box to allow external user to use unified logon protocol OpenID. If you select this check box, the 1C:Enterprise authentication check box can be cleared.

Adding external users to groups

If the application supports user groups, you can add the external user to one or several user groups immediately during user creation procedure.

  • Click Groups on navigation panel. Select one or several groups via check boxes.
  • To clear all selections quickly, click Exclude from all groups.

External user access rights 

Access rights can also be set up by the external user list manager, to a limited degree (excluding administrative rights). You can use one of the following methods to set up access rights:

  • Click Access rights on navigation panel to add an external user to one or more access groups. Select the required groups from list.
  • In the Allowed actions (roles) list, you can assign one or several roles to an external user, thus defining their personal access rights. Some roles may correspond to job functions or duties, some - to more specific functions. Select the roles you need to assign via check boxes.

Allowing application logon (for administrator only)

To allow an external user to logon to the application, select the Access to infobase allowed check box and provide the required information: logon name, password, and other authentication parameters. 

Managing external user settings

Individual settings of each user are saved by the application for further use. To view, copy, or clear external user settings (clearing may be required, for example, when invalid settings prevent normal application use), click Settings on navigation panel.

Application warnings

If external user properties differ from the Infobase user properties, the application will display a warning including the list of differences. This list should be saved in order to reconcile the differences.
