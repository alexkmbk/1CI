This list is used to quickly and easily reconcile the list of infobase users in Designer mode with Users and External users lists.

To access the list, use the All actions - Infobase users command in user list (or external user list) menu.

When you create or modify a user (external user), a corresponding infobase user will be created or modified automatically. However, in certain cases, discrepancies may occur between the infobase user list and the application user list. This list can be useful when you need to:

  • Identify invalid infobase users. 
    • The application compares infobase users against the application user lists automatically.
    • If an application user was added, modified, or deleted in Designer mode, the corresponding infobase user will be disabled.
    • Infobase users that cannot be mapped to any member of application user lists are marked in red.
  • Find discrepancies between infobase user list and the application user lists:
    • If an infobase user was added via Designer (or any other inappropriate method), you will be asked to map this user with an application user list member manually, or Delete them;
    • If an infobase user was deleted in Designer mode (or any other inappropriate method) but they are still referenced by a user list member, you will be asked to modify the user list member manually in order to clear the reference, or to create a new infobase user to be referenced by the user list member;
    • If an infobase user name was modified via Designer (or any other inappropriate method), you will be asked to modify the user list member manually in order to restore the correct infobase user name.
  • View the infobase user list (also available in Designer mode).

The list displays the following information:

  • Full name - full name of infobase user. In case of discrepancies, can differ from the full name of the Users (External users) list member;
  • Name (for logon) - in case of discrepancies, can differ from the logon name of the Users (External users) list member;
  • 1C authentication - takes on "Yes" value if check box is selected, or remains blank otherwise;
  • OpenID authentication - takes on "Yes" value if check box is selected;
  • OS authentication - takes on "Yes" value if check box is selected.

Additional data can be displayed via the All actions - Customize form command:

  • OS user - OS logon name, also used for application logon.
  • Mapped to external user - takes on "Yes" value if this is an external application user;
  • Added in Designer - takes on "Yes" value or remains blank if False;
  • Modified in Designer - takes on "Yes" value or remains blank if False;
  • Deleted in Designer - takes on "Yes" value or remains blank if False.

Viewing invalid records only or whole list 

  • To view only unmapped records that require manual processing, select the Show only users added, modified, or deleted in Designer mode check box. To view the whole infobase user list, clear this check box.

Mapping users

To reconcile the user lists, the following options are available:

  • Map - manually map an infobase user to an existing application user, as follows:
    • select a User or External user value for mapping and click OK. 
    • select a user from the corresponding list and click Select, or double-click the user.
    • The User (External user) window will open; make the required edits in this window, then click Save and close. This will restore the correct mapping between the lists.
  • Map with new user - manually map an infobase user with a new application user, as follows:
    • select a User or External user value for mapping and click OK. 
    • Create a new application user (external user). This will restore the correct mapping between the lists.
  • Clear mapping - clear the automatically generated mapping between infobase users and application users prior to creating a manual mapping. This function is only demanded in rare cases of mapping performed incorrectly (for example, during infobase update). We do not recommend to clear user mapping for any other reason.

Viewing mapped users

  • Select an infobase user. Click Go to user to view the application user (external user) mapped to the selected infobase user.
