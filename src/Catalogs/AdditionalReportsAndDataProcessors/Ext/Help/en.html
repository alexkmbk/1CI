This list is used to store additional reports and data processors in an application.

You can connect and utilize a variety of additional (external) reports and data processors in 1C:Enterprise mode. This extends the application's functionality without making any changes to the application, which can be useful when you need to custom-fit the application to specific purposes without losing the technical support rights.

The additional reports and data processors are created in Designer mode. If you want an additional report or data processor created, you need to contact the developers with this request. Alternately, you can download pre-made reports from the Internet.

To enable additional reports and data processors, use the Print forms, reports and data processors command in the Administration section and select the corresponding check box. This will enable the Additional reports and data processors command. The check box is selected by default. You can also access the list by executing a related command on navigation panel of the Integrated subsystems section.

The list includes:

  • Description - full name of a report or data processor.
  • Type - each additional report or data processor is described by its type as assigned by developers. Available functionality is governed by the report type.
  • Publication - visibility of additional reports or data processors, color-marked. Additional reports and data processors in debug mode are marked in red and accessible by administrator only. Disabled additional reports and data processors are marked in grey. Additional reports and data processors marked for deletion are disabled and marked both in grey and with a special icon.
  • Version - version number of report or data processor (specified by developer).
  • Manager - application user that was assigned as a manager responsible for this report or data processor. This user (by default, Administrator) is responsible for monitoring the additional report/data processor version changes, updating to latest versions, and more.

You can execute the All actions - Change form command to add the following columns to the form:

  • Object name - includes type and other information. This field is filled in by developer and displayed in the appliction by default.
  • Permission compatibility mode - application version used with the report or data processor.
  • Comment - detailed information on the additional report or data processor;
  • Use option storage - additional report property. Takes on the Yes value if the additional repost stores data in the application option storage;
  • Use for list form - takes on the Yes value if the command calling the additional report or data processor is located in list windows;
  • Use for object form - takes on the Yes value if the command calling the additional report or data processor is located in list item windows;
  • Information - description of the additional report or data processor as provided by developer;
  • File name - name and extension of file where the additional report or data processor is stored;
  • Safe mode - takes on the Yes value is the additional report or data processor is executed in safe mode. As a rule, data processors should be executed in safe mode. This is a safeguard against untrusted code being executed on server side. However, certain data processors may require execution in unsafe mode. In this case, a warning will be displayed. These data processors should only be obtained from trusted sources (vendors) or manually audited before execution. For more information on safe mode, refer to the 1C:Enterprise platform documentation.

Inserting additional reports or data processors

  • You can select an external report or data processor file on your computer (specify full path to it) and configure its parameters by executing the Create command. The data list may vary depending on the purpose of the selected report or data processor. If an invalid file was specified, an error message will be displayed (example: the selected additional data processor cannot be used with the current application version). 

Grouping additional reports and data processors

For your convenience, extensive lists of additional reports and data processors can be grouped by a variety of attributes. For this, fill in the required number of groups:

  • Click  and fill in the group name. You can make a new group subordinate to another group. For this, select the group from list in the Parent field.
  • Drag the required additional reports or data processor to this group.

Filters

The following filters can be enabled to facilitate your navigation of extensive lists of additional reports and data processors:

  • the Type field - click Show all, select an additional report or data processor type to be used in filter (for example, Reports). 
  • the Publication field - click Show all, select a status to be used in filter (for example, Used).
  • Both Type and Publication fields - in extensive lists, both filters can be combined.

Loading from file

To update a data processor in internal application storage, use the Load from file command. Specify file path and click Open.

Saving to file

To export a data processor or report from storage to computer file, use the Save to file command. Specify folder and click Save. File name will be generated automatically but can be changed if necessary.

Additional report and data processor types

Additional reports and data processors are categorized into the following types:

  • Global (common) reports and data processors - these are used independently from the application data. Commands for working with global reports and data processors are located in the application sections. Global reports and data processors can be used to load data into the application, update or clear the full-text search index, etc. They contain commands that can only be called from command interface sections. The commands are independent of any applied objects. The list of sections where commands can be placed is predefined by the developer, included in additional report or data processor file, and cannot be modified.
  • Assignable additional reports and data processors - these are used with specific application lists or documents, single or multiple. The list of objects to which such a report or data processor can be assigned is predefined by developed and embedded into text of the report or data processor. When assigning a report, you can only select targets from this list. This type is further broken down into subtypes:
    • Report - assignable additional report. It can be used to create a report on a specific applied object.
    • Object filling - assignable additional data processor that can be used to automatically fill in or clear single or multiple list items.
    • Print form - assignable additional data processor that can be used to create new print forms for a document.
    • Related object creation - assignable additional data processor that can be used to fill in lists quickly by copying list items.
