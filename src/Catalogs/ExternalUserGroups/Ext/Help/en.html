This list is used to manage external user groups.

External user groups can be convenient when a large number of external users work with the application: 

  • to group large numbers of external users (for example, during external user selection);
  • to quickly set up access rights for all external users in a group.

Groups are created as a flat list (all groups in the list are independent), or as a hierarchical structure. In a hierarchical list, external users of a subordinate group also belong to the parent groups.

Predefined group All external users contains all external users. This group is specified as current group by default.

If roles are assigned directly, you can assign roles both to single external users and to external user groups.

Grouping external users

External user groups can be created, for example, based on their relation to applied objects:

  • Counterparties
    • Trusted Partners
    • Major customers
    • Major vendors
    • Poll responders
  • Individuals.

You can create external user groups containing only members of a specific type, such as poll responders. This approach facilitates access rights setup.

Creating groups

  • Click Create and fill in the required data.
