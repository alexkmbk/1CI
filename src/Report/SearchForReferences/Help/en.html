This report is used to view information on duplicate list item usage in application documents.

Routine user operations may sometimes result in unintended creation of duplicate items, interfering with accounting procedures. Documents start referring to different copies of the item, which leads to incorrect debiting and other errors.

Before deleting the duplicate items, you need to delete from the documents all references to these items.  This report indicates which application documents contain references to duplicate items.

The report can be accessed from any application lists via All actions - Usage instances command.

The report displays the following information:

  • Description and Code of each selected list item;
  • Usage instances - document names, numbers and dates;
  • Occurrence count.

Generating report

  • Select one or several list items, and execute the All actions - Usage instances command. The report will be generated automatically.
  • The report is grouped by selected list items. For each item, the list of application documents containing references to this item will be displayed, as well as occurrence count for each document and total occurrence count for the item.
  • To view a list item or a document referencing the item, double-click the corresponding report cell.
