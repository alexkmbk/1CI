This form is used to find and merge duplicate data.

The list of items to be merged displays:

  • Description,
  • Code,
  • Usage count - this field displays item occurrence count, or Not used message.

Merging items

  • The first item in the list is marked for merging by default.
  • Select the "correct" item you want to keep using, and click Merge to. This will mark the selected item with icon as item merge target.
  • Infobar at the bottom of the list displays total number of items to be merged and description of the merge target item. All other items will be marked for deletion (via a hyperlink editable by administrator).
  • Click Merge.

  • After the merging, only the target item will remain in the application; all other selected items will be replaced with the target item and marked for deletion (if you have administrative rights, you can permanently delete these items as well).

  • If any errors are encountered during the merge procedure, a list of items that failed to be processed will be displayed, complete with description of failure causes. For example, merging will fail if the items belong to different list groups with different accounting.

  • To continue with merging, you need to eliminate the error cause by changing item content.

Changing list item content

  • If the merge list does not display the items you want, you can  Add new item. Select an item from the item list, or add a new item.
  • If the merge list contains more than 2 items, you can  Delete excessive items from the list.

Viewing items

  • Select an item and click Open.

Viewing references to list items

Click All usage instances hyperlink to view Usage instances report for all list items.

Configuring item deletion (for administrators only)

If you have administrative rights, you can permanently delete any items marked for deletion by changing the hyperlink at the bottom of the list from marked for deletion to deleted permanently.
