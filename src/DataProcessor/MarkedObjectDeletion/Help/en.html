This form is used to permanently delete objects marked for deletion.

While working with applications, users may need to mark certain objects for deletion (for example, objects created by mistake). Objects marked for deletion can be permanently deleted by a user with administarive rights.

To delete a marked object, execute the Deletion of marked objects command located in the Administration - Support and maintenance section.

During deletion procedure, reference integrity control is performed. If an applied object (document, catalog item, etc) is referenced by other objects, it cannot be deleted. Keep in mind that deletion of the marked objects is irreversible.

When deleting the marked objects, two methods are available (selectable by radio button):

  • Delete all marked objects automatically - in this mode, all objects marked for deletion will be deleted permanently.
  • Delete objects selectively - in this mode, the user selects which objects to delete.

Delete all marked objects automatically 

  • Click Delete.
  • In this mode, all objects marked for deletion will be identified and deleted. Once the procedure is complete, a notification containing number of deleted objects will be displayed. User intervention is only required when a marked object is referenced by other objects and cannot be deleted.

Delete objects selectively

  • Click Next.
  • In this mode, a list of objects marked for deletion will be generated.
    •  Objects in this list are grouped by type. All check boxes are selected by default.
    • If you do not want to delete some of the marked objects in the list, clear the corresponding check boxes.
    • To view an object and decide whether it should be deleted, double-click an object, or select an object and click  Change.
  • To delete the selected objects permanently, click Delete.
  • Once the procedure is complete, a notification containing number of deleted objects will be displayed.
  • If any of the marked objects were referenced by other objects, the application will generate a warning and display a list of marked objects that were not deleted due to references, as well as a list of objects referencing them.

How to proceed when an object cannot be deleted

The list of objects that could not be deleted consists of two sections: object list on the left, and the Usage instances list on the right. 

  • To delete these objects permanently, you need to:
    • Double-click an object preventing deletion, or select it and click  Change, to open the object.
    • Delete all references to the marked object manually.
    • Or replace the references that prevent deletion. Click Replace everywhere where found to proceed to item replacement. To replace a reference, select a replacement item from the list.
    •  Mark to delete all objects associated with the item to be deleted (Usage instances on the right).
    •  Clear deletion mark from the marked object.
  • Then click Retry deletion to retry object deletion.
  • Click Restart to select a different deletion method.
  • Click Close to cancel the deletion procedure and go back to the application.
