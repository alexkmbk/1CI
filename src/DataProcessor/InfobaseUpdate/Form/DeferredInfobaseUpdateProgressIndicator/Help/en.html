This form is used to display progress and view results of deferred infobase updates.

When performing infobase version updates, you may choose to have some of the data updated in deferred mode, i.e. in background, without service interruption. In this mode, the documents that have not been processed yet will become temporarily unavailable. Any attempt to open such a document will fail and display a message informing the user that the document is temporarily blocked until infobase version update is complete. Some reports may also become unavailable for the duration of the update. Any attempt to create such a report will display a warning informing the user that reports generated during the version update procedure may contain invalid data.

The form, as well as the CommonForm.ApplicationReleaseNotes/Help">Update details (What's new) window, opens automatically as you log on to the application after infobase update.

You can also access information on the deterred update results by executing the Update results and additional data processors command located in the Administration - Support and maintenance section. Alternately, you can access this information by clicking the Additional data processors required hyperlink located in the What's new window. Once data processing starts, the hyperlink will change to Additional data processors in progress.

Information on completed infobase updates

Update version, update date and time, and update procedure duration are specified.

Viewing event log

To view the infobase update events in the event log, click the Technical details on update results hyperlink.

Executing data processors

  • Until a data processor is started, the hyperlink text is set to List of data processors. Click the hyperlink to view the list of additional data processors.
  • Click Execute to start data processing.
  • While a data processor is running, the hyperlink text displays the number of completed data processors.
  • After all data processors are completed, completion date and time are displayed. The hyperlink is set to All update procedures completed successfully, followed by the total number of completed procedures in brackets.

Setting data processing schedules

Data processors often take a long time to complete (from several minutes to hours, depending on volume of data stored in the application and computer performance). Moreover, while data processors are running the user experience may suffer. We recommend that you schedule the data processors to run during the minimum user activity periods, by setting up the Deferred infobase update scheduled job.

  • To set up execution of the Deferred infobase update scheduled job, click Set schedule.
  • You can also set up the scheduled job via common list of scheduled and background jobs, by executing the related command located in the Administration - Support and maintenance section.
