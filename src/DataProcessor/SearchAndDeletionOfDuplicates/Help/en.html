This form is used to identify and delete duplicate items that may be present in any application lists. It is intended to be used by application administrators.

Routine user operations may sometimes result in unintended creation of duplicate items (example: user was unable to find a specific item due to incorrect search request, and decided to create the item once again), interfering with accounting procedures. Documents start referring to different copies of the item, which leads to incorrect postings and invalid balance calculations. To avoid this, any duplicate items must be found and deleted.

The application includes functionality for finding duplicates, marking them for deletion, and replacing references to duplicate items with references to the "original" item.

The Search and deletion of duplicates command is located in the Administration - Support and maintenance section.

Searching for duplicates

  • In the Search in field,  select an application list.
  • In the Filter field, specify list item filter conditions. To do this, follow the All items hyperlink (default value) and specify Filter rules.
  • Select Field to be used as filter. Specify search value. Select comparison operation in the Comparison type field and specify Value for comparison. Click OK to go back to the duplicate search form. The hyperlink will change to display the filter conditions you selected.
  • To delete a filter, click
  • Predefined Duplicate search rules will be automatically entered into the Compare field, in hyperlink format. To add a new item to the rules list, click the hyperlink, select a check box corresponding to the desired rules item, and click Select. The hyperlink will change to display the updated information.
  • When the duplicate search rules are complete, click Find.
  • If no duplicates were found, you can refine search conditions and click Retry search.

      Viewing duplicate list

      • The list of found duplicates is divided into two sections:
        • The left section contains the list of found items, displayed as a tree and grouped by name.
        • The right section displays information on the currently selected items:
          • for item groups (on per-item basis) - report on the number of duplicates found for the selected item.
          • for items - usage report (list of application documents using the item, or message "Not used").
      • One of items used by the application is automatically marked as  Original.
      • To mark a different item as original, select it and click  Set as original.
      • Duplicate items in the list are marked with flags. 
      • To view the selected item, click Open. Or double-click the item.
      • Using buttons Expand duplicate group orCollapse duplicate group, you can control visibility of item groups in the list.
      • Infobar at the bottom of the list displays the number of items and the number of found duplicates.
      • If you want to improve the list of found duplicates, you can refine search conditions and click Retry search.

      Deleting duplicates

      • If you agree with all conditions displayed on the list infobar, click Delete duplicates. All duplicates will be marked for deletion, and replaced with the selected Original throughout the documents.
      • To exit the form without deleting any duplicates, click Close. Search conditions and the list of found duplicated will be lost.
      • To delete the duplicates permanently, use the Marked object deletion command in the Administration - Support and maintenance section.

      Viewing references to list items

      Click the All usage instances hyperlink to view the Usage instances report for all list items.
