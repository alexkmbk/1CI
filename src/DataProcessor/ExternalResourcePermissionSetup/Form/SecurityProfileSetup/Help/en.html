This panel is used to set up usage of infobase security profiles for 1C:Enterprise server cluster. The security profiles control how the application uses external (relative to 1C:Enterprise server cluster) resources.

They are accessed from Administration - Support and Maintenance - Security section.

Security profile usage mode

You can select security profile usage mode for an application via radio button:

  • Security profiles are not used for this Infobase 
    • The application does not notify administrator that external resources must be added to security profiles.
      • If security profiles are enabled in a server cluster and this mode is selected, any attempt to access external resources not permitted for this cluster server will result in application error.
      • If Full access to external reports and data processors check box is unchecked in security profile, the application will not start.
  • Security profiles for this infobase are set up by administrator via cluster console
    • The application does not notify administrator that external resources must be added to security profiles
    • The administrator sets up permissions in security profiles via cluster console. 
      • The list of permissions that must be added to security profile is available to the administrator in a report accessed by clicking Required permissions hyperlink on the panel.
      • Safe mode profile check box must be selected for the infobase profile.
      • In Infobase security profile field, fill in the name of security profile used for this infobase.
      • If Full access to external reports and data processors check box is unchecked in security profile, you need to select Can be used as security profile check box and specify the security profile name in Infobase security profile field on this panel. Otherwise, the application will not start.
  • Option to automatically add permissions to security profiles when necessary
    • When you switch to this mode, a new security profile will be created (and assigned to Infobase) that permits access to all external resources required for the application to operate correctly.
    • Permissions in security profiles are managed by the application.
    • Profile management functionality is accessible from the application.
    • If the administrator needs to grant access to an external resource while editing the application settings, a dialog will be displayed with permission list.
      • If the administrator agrees to grant these permissions, they will need to enter the cluster authentication parameters and click Next.
      • Permissions in security profiles are granted automatically.

Required permissions

This command cannot be used if Profile security usage mode is set to Security profiles are not used for this infobase.

  • Click Required permissions to display a report listing external resources that must be permitted in security profiles for the application to operate correctly.

Infobase security profile

This field is read-only if Profile security usage mode is set to Option to automatically add permissions to security profiles when necessary.

If Profile security usage mode is set to Security profiles for this infobase are set up by administrator via cluster console, and Full access to external reports and data processors check box is unchecked in security profile - you need to select Can be used as safe mode profile check box and specify the security profile name in this field. Otherwise, the application will not start.

  • In Infobase security profile field, enter the name of security profile to be used (in 1C:Enterprise server cluster).

Restoring security profile settings

This command can only be used when Profile security usage mode is set to Option to automatically add permissions to security profiles when necessary.

This command fully rebuilds the infobase security profiles considering external resources required to run the application with current settings. This can be useful, for example, after the infobase was migrated between server clusters.

  • Click Restore security profile settings.
