This report is used to review information on all application users and external users, including logon settings (infobase user properties).

It can be helpful when you need to perform mass analysis of user logon settings (logon names, authentication types, etc).

To access the report, use the All actions - User (external user) information command in the Users (External users) list

The report contains the following information:

  • Full user name;
  • Can log on to application - field can take on "Yes" value or remain blank
  • Name (for logon);
  • Standard authentication - field can take on "Yes" value or remain blank;
  • OS authentication - field can take on "Yes" value or remain blank;
  • OS user - user name for OS logon;
  • OpenID authentication - field can take on "Yes" value or remain blank;
  • Run mode.

User filters

To go to Settings tab, click Settings and other reports. Click the corresponding check boxes to select users by specific criteria.

The following user filters are available:

  • Invalid
  • Marked for deletion
  • No matching infobase user
  • Can log on to application
  • Standard authentication
  • OS authentication
  • OpenID authentication

Report options

To go to Other reports tab and view the list of available report options, click Settings and other reports. Three report options are available:

  • User information - accessible from Users list;
  • External user information - accessible from External users list;
  • User and external user info - general report on all application users, containing Is external user field used for selecting and filtering external users. The field can take on "Yes" value or remain blank.
