Two complementary infobase update methods are available for your convenience:

  • using exclusive processors:
    • all users are locked out of the application.
    • this method is used to update data that must be fully processed prior to the users logging back on.
      • the less data volume, the shorter downtime.
  • using deferred processors:
    • user are free to log on and work.
    • this method is used to update the infobase "data archive" (i.e. data not actively accessed by the users, or data that can be accessed by the users while the update is running).

Once the application update (in Designer mode, or using the Configuration update wizard available in basic versions) is complete, the application must be started for the first time on behalf of the administrator. During this first run, the application will finalize certain update procedures, which could take a long time (depending on the update scope and computer performance).

We recommend that you terminate all user sessions before running the application for the first time after update. This is necessary for correct application update.

If any user sessions are still open, you can terminate the active user sessions in file mode, and lock the application temporarily (if you attempt to start the application in client/server mode, the start will be canceled and "active user sessions detected" message will be displayed). Click the hyperlink to view the list of active users and ask them to log off, or click Terminate sessions and retry to continue with the infobase update.

After that, the application will start preparing to terminate the active user sessions. All active users will be asked to log off. The application will then lock as scheduled, and version update procedure will resume.

However, the application will not shut down; it will continue running displaying Application update message on an empty desktop. The message includes the updated infobase version. The indicator displays infobase update progress - both visually and as a percentage. To view detailed information on the infobase update progress, click the Technical details on update progress hyperlink located in the event log.

If an obsolete platform version is used, the application will display a message requiring platform version update and lock the desktop.

Once the infobase update is complete, the application will restart automatically. However, application desktop will be locked; to unlock it,  you will need to confirm that the update was legitimate. After confirming this, you are free to start working with the application.

If any issues arise during the update, the application will display an error message asking you to contact technical support. To view the update error messages, click the More details in event log hyperlink. Users will not be able to resume working in the application.

Once the application update is complete, you can view the CommonForm.ApplicationReleaseNotes/Help">change notes for the new version (available in the What's new window), view update-related technical events recorded in the event log, and start or monitor deferred data processing (if available in the current application version). Afterwards, the What's new window will also be accessible via desktop. The application will unlock for all users.

Features and advantages of deferred infobase updates

  • In client/server mode, application update may require additional data processors that should be executed in background to reduce downtime. Users can start working with the application before the processing is complete. White the update procedure is running, certain application functionality may be temporarily unavailable.
  • In this mode, the documents that have not been processed yet will become temporarily unavailable. Any attempt to open such a document will fail and display a message informing the user that the document is temporarily blocked until version update is complete. Some reports may also become unavailable for the duration of the update. Any attempt to create such a report will display a warning informing the user that reports generated during the version update procedure may contain invalid data.

Monitoring deferred updates

If application update includes deferred background data processing, the administrator can monitor the deferred version update progress by executing the Update results and additional data processors command located in the Administration - Support and maintenance section. Alternately, you can access this information by clicking the Additional data processors in progress hyperlink located in the What's new window.
