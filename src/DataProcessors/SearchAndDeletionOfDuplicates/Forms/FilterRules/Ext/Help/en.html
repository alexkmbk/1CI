This form is used to create list item filters.

Applying filter conditions

  • Click Add new item. 
  •  Select a Field from the list to be used as filter.
  • In the Comparison type column, Select a comparison operation. Lists may include generic (Equal, Not equal, Greater, Less, etc) or more complicated operations:
    • The In list comparison operation can be used to specify the list of allowed filter values. The Not in list comparison operation is inversion of In list.
    • Contains and Not contains operations are used for filters based on partial field content (a sequence of characters found or not found anywhere in the field value).
    • Filled and Not filled operations are used to determine if field contains any value or not.
  • Specify Value. For this, depending on the selected field and comparison type,   Select a value from the list, enter a value, or create a value list (used for In list and Not in list).

  • To delete a filter, click .

  • To confirm and return to the application, click OK.

Grouping conditions

To refine your filter, you can apply multiple conditions simultaneously. The conditions can be reordered by dragging or by clicking  and .

  • If a filtering rule includes multiple conditions that are ungrouped, the And logical operation will be applied by default.
  • You can apply And, Or, Not logical operations to any selected conditions. For this purpose, Group types are implemented: And group, Or group, Not group.
  • To group multiple conditions together, select one or seleral conditions and click Group conditions.  By default, all selected conditions will be added to And group.
  • You can change the group type if needed. For this, double-click the Group to open the Group type selection cell.   Select a Group type you need. You can also modify group types manually.
  • Drag additional conditions to the group, or create new conditions within the group.
