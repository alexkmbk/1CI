From time to time, application administrator may need to monitor which events occurred in the application during a specific period, or which activities were performed by a specific user. All such information can be obtained from the event log.

Event log contains the list of user activities related to data modification, scheduled job execution, connecting and disconnecting from the application, and more. Event log settings can be configured in Designer.

To access event log, use the corresponding command in the Administration - Support and maintenance section.

All application events are grouped into the following types and marked in event log correspondingly:

- Error;
- Warning;
- Information;
- Unavailable field details;
- Note.
Each event is registered on a separate line.

The event list includes:

  • Date, time - date, time, and event type icon;
  • User - full user name
  • Computer - network ID of the computer;
  • Application - Thin client, etc; 
  • Session - session number; 
  • Event - event type;
  • Comment - brief description of the event;
  • Transaction status - Committed, etc;
  • Transaction ID - transaction ID (transaction date, time, number);
  • Metadata - applied object type and description: list, document type, etc;
  • Data - applied object indication: name of specific list item, document, etc;

  • Data presentation - applied object indication.

To customize visibility and placement of log columns, click All actions - Customize form.

Refreshing event list

The event list is not refreshed automatically. To display events that occurred in the application after the log was opened, click  Refresh current list (or press F5). 

Event count

The event log displays a predefined number of records specified in the Show field (200 by default). You can select a predefined value from list (200, 500, 1000, 10000 events), or enter a custom value, which will be saved for further use.

Filters

You can use event log filters to display only those events you need.

  • In the Criticality quick filter field, select Errors or Warnings to display only events of a specific type. By default, All events are displayed in the log.
  • To set event view date interval for the event log, click  Set view date interval. Or double-click the content of the Date, Time column.
  • Select data in a log column and click Set filter by value in current column. The filter will only display events containing the selected data. This command is also available via context menu accessed by right-clicking.
  • To filter events by one or several criteria, click Set filter, or click the hyperlink in the Filter field. If any filters (including quick filters) are enabled, the hyperlink text will list the filter criteria. Otherwise, it will display the Not set text. The following filters can be set:
    • The Interval field - select period, specify Beginning of period and End of period;
    • The Importance field - use check boxes to select one or several event types (Error, Warning, Information, Note) for filtering. All check boxes are selected by default;
    • The Event field - use check boxes to select one or several list values. All check boxes are selected by default;
    • The User field - use check boxes to select one or several list values. Only users registered in event log are listed. All check boxes are unchecked by default;
    • The Applications field - use check boxes to select one or several list values (Designer, Thin client, Background job). All check boxes are unchecked by default;
    • The Computers field - use check boxes to select one or several list values. All check boxes are cleared by default;
    • The Sessions field - enter session numbers separated by semicolon;
    • The Data field group - select one of the available filters:
      • Metadata field - use check boxes to select one or several list values (the list is sorted by data type). Select list or document name. such as, "Bank classifier catalog"
      • Data field - select items from application lists:
        • select Data type, such as "Bank"; 
        • In the corresponding application list (for example, "Bank classifier"), select an applied object (for example, "Bank of America");
    • The Data presentation field - enter search data manually or copy it from a list;
    • The Transaction field group - use check boxes to select one or several Transaction status values (Not applicable, Committed, Unfinished, Rolled back). All check boxes are selected by default. Transaction ID field can also be used as a filter.
    • The Others field group - use check boxes to select Working servers, Ports, Sync ports. The Comment field can also be used as a filter.
  • Once all fields to be used as filters are selected, click OK. The event filters will be enabled, and the hyperlink will display the selected filter criteria.
  • Only the records conforming to the enabled filter criteria will be displayed. Total number of the displayed records is limited by current value of Show field.
  • To clear all filters (including quick filters), click Clear filter or execute the All actions - Clear filter command. This command is also available via context menu accessed by right-clicking.

Viewing events

  • To view detailed information on an event, select the event from the list and click  Open. This command is also available via context menu accessed by right-clicking.
  • You can also double-click the corresponding record to view the event details. 

Viewing applied objects

If an applied object is related to an event, you can open this object for detailed viewing.

  • Select data in the Data or the Data presentation column for the corresponding event and click  Open data for viewing. This command is also available via context menu accessed by right-clicking.
  • You can also double-click the Data or the Data presentation column to view the data.
